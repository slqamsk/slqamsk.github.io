🔧 Настройка GitHub Pages для отображения Mermaid-диаграмм из Markdown-файлов

📋 Постановка задачи

GitHub позволяет хранить в репозитории Markdown-файлы (.md), а при включении GitHub Pages — автоматически публиковать их как веб-страницы. Встроенный генератор статических сайтов Jekyll преобразует Markdown в HTML и применяет к нему выбранную тему оформления.

Однако диаграммы, написанные на языке Mermaid (блоки ```mermaid), по умолчанию отображаются как обычный текст кода. Эта инструкция описывает, как настроить репозиторий так, чтобы Mermaid-диаграммы в опубликованных .md файлах отрисовывались в виде графических схем, при этом остальные файлы (готовые HTML, изображения, стили) оставались нетронутыми.

📁 Примечание о структуре папок: В данной инструкции все файлы Jekyll (_config.yml, _layouts/) лежат в папке docs/, потому что в настройках GitHub Pages выбран источник «Deploy from a branch → /docs».

Это не обязательное требование. Вы можете выбрать источник «/ (root)» в настройках репозитория (Settings → Pages → Source). В таком случае все указанные файлы нужно будет положить прямо в корень репозитория, а не в папку docs/. Логика работы от этого не меняется.
💡 Полезное расширение для VS Code: Для локального предпросмотра Mermaid-диаграмм в том виде, как они будут выглядеть на GitHub, установите расширение «Markdown Preview Enhanced» (автор: Yiyi Wang). Оно позволяет видеть отрисованные диаграммы прямо в редакторе при редактировании .md файла.

📁 Итоговая структура репозитория (для источника /docs)

ваш-репозиторий/
├── docs/
│   ├── _config.yml
│   ├── _layouts/
│   │   └── default.html
│   └── courses/
│       └── ваш-файл.md
└── (остальные файлы)

🚀 Пошаговая настройка

Шаг 1. Создайте файл конфигурации Jekyll

Создайте файл docs/_config.yml (если папки docs/ нет — создайте её) со следующим содержимым:

theme: minima

Строка theme: minima указывает Jekyll использовать тему Minima, которая позволяет переопределять шаблоны страниц. Никаких дополнительных настроек не требуется.

Шаг 2. Создайте шаблон страницы

Создайте папку docs/_layouts/, а внутри неё файл default.html со следующим содержимым:

{% raw %} 
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    {% seo %}
    <title>{{ page.title | default: "Сергей Слесарев - курсы по тестированию" }}</title>
    <link rel="stylesheet" href="/assets/css/style.css">
    <script type="module">
        import mermaid from 'https://cdn.jsdelivr.net/npm/mermaid@11/dist/mermaid.esm.min.mjs';
        mermaid.initialize({ 
            startOnLoad: false,
            securityLevel: 'loose'
        });
        
        window.addEventListener('DOMContentLoaded', () => {
            document.querySelectorAll('code.language-mermaid').forEach((block) => {
                const div = document.createElement('div');
                div.className = 'mermaid';
                div.innerHTML = block.innerHTML;
                block.closest('pre').replaceWith(div);
            });
            mermaid.run();
        });
    </script>
</head>
<body>
    {{ content }}
</body>
</html>
{% endraw %}

Этот шаблон содержит скрипт, который находит сгенерированные Jekyll блоки кода Mermaid и преобразует их в формат, понятный библиотеке отрисовки. Также в нём подключен SEO-плагин {% seo %} для автоматической генерации мета-тегов.

Шаг 3. Подготовьте Markdown-файлы

В каждый .md файл, в котором должны отображаться диаграммы, добавьте в самое начало блок Front Matter:

---
layout: default
title: Заголовок страницы (можно не указывать)
---

После закрывающих --- оставьте одну пустую строку, а затем пишите ваш обычный Markdown-контент с диаграммами в блоках ```mermaid. Всё содержимое файла после Front Matter остаётся без изменений.

Пояснение: Строка layout: default указывает Jekyll применить к этому файлу созданный нами шаблон default.html. Строка title: опциональна — если указать, заголовок попадёт в <title> страницы; если нет — будет использовано значение по умолчанию.

🔧 Как это работает (техническая суть)

  1. Jekyll при сборке находит .md файлы с Front Matter и применяет к ним шаблон default.html.
  2. Блоки ```mermaid превращаются в <pre><code class="language-mermaid">...</code></pre>.
  3. Библиотека Mermaid по умолчанию ищет элементы с классом mermaid и не понимает формат Jekyll.
  4. Скрипт-преобразователь в шаблоне находит все code.language-mermaid, пересобирает их в <div class="mermaid"> и запускает отрисовку.
  5. Параметр securityLevel: 'loose' разрешает Mermaid обрабатывать код из нестандартных источников.
  6. Все файлы без Front Matter (готовые HTML, изображения, CSS) копируются в собранный сайт без изменений.

❓ Часто задаваемые вопросы

1. Где в логах GitHub Actions смотреть, что тема и шаблоны применились правильно?
На вкладке Actions вашего репозитория найдите последний запуск workflow pages-build-deployment. Раскройте шаг «Build with Jekyll» и проверьте две ключевые строки.

Проверка темы:

Theme: minima

Если вместо minima вы видите другую тему (например, jekyll-theme-primer) — значит ваш _config.yml не найден или проигнорирован. Проверьте, что файл лежит в правильной папке (docs/ или корень репозитория в зависимости от настройки Pages).

Проверка шаблонов:

Layout source: site

Запись site означает, что Jekyll использует ваши пользовательские шаблоны из папки _layouts/. Если бы шаблоны не были переопределены, вы увидели бы Layout source: theme (используется встроенный шаблон темы).

В логе вы увидите такие строки для каждой обрабатываемой страницы, например:
Rendering Layout: courses/ai-qa-draft.md
Layout source: site

2. Почему скрипт в шаблоне — это необходимый «костыль»?
Jekyll и Mermaid «говорят на разных языках». Jekyll считает диаграмму кодом для подсветки синтаксиса и оборачивает в <code class="language-mermaid">. Mermaid ищет <div class="mermaid">. Скрипт — это «переводчик», который преобразует одно в другое на лету в браузере.

3. Что делать, если диаграммы всё равно не отображаются?

  • Проверьте вкладку Actions — сборка должна быть зелёной ✅.
  • Откройте консоль браузера (F12 → Console) — нет ли там ошибок JavaScript (красные сообщения).
  • Убедитесь, что в .md файле есть layout: default внутри блока ---.
  • Проверьте, что файлы _config.yml и _layouts/default.html лежат в правильной папке (в docs/ или в корне — в зависимости от настройки Pages).
  • Дождитесь полной сборки сайта (обычно 2-3 минуты) и очистите кэш браузера (Ctrl+F5).

4. Повлияет ли эта настройка на мои готовые HTML-файлы?
Нет. Jekyll обрабатывает только файлы, в начале которых есть блок --- (Front Matter). Все остальные файлы (HTML, CSS, изображения, текстовые файлы) копируются в собранный сайт один-в-один без каких-либо изменений.

5. Что делает SEO-плагин {% seo %} и нужен ли он?
Этот плагин автоматически генерирует мета-теги для поисковых систем и социальных сетей (заголовок, описание, Open Graph-разметку, канонический URL). Если вы планируете делиться ссылками на эти страницы, SEO-теги обеспечат красивое и информативное превью. Если страницы используются только локально или внутри команды — можно удалить строку {% seo %} из шаблона, на работоспособность диаграмм это не влияет.
