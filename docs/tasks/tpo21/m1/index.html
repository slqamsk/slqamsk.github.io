📘 О системе

Данное приложение демонстрирует разницу между символьной (лексикографической) и числовой (математической) сортировкой на примере списка клиентов.

  • Номер клиента – сортируется как строка (посимвольно). Это нужно, чтобы порядок в системе совпадал с порядком файлов в папках (например, клиент с ID 10 идёт после ID 1, но до ID 2).
  • Расстояние до клиента – сортируется как число (по значению). Это нужно для маршрутизации и приоритезации.

Перейдите на страницу «Данные», чтобы увидеть таблицу и попробовать сортировку.

На странице «Генерация тестов» вы можете добавить специальные наборы данных или очистить все записи.

Все данные автоматически сохраняются в localStorage браузера.

📊 Данные клиентов

Номер клиента → сортируется как строка (символьная)
Расстояние → сортируется как число (математическая)
№ Номер клиента символьная Название Расстояние, км числовая
🔍 Как это работает:
• При сортировке по номеру клиента числа сравниваются как строки: "10" < "100" < "1000" < "2" — все начинающиеся с "1" идут вместе.
• При сортировке по расстоянию сравнение числовое: 1 < 2 < 3 < ....
• Нажмите на заголовок колонки, чтобы переключить сортировку.

⚙️ Генерация тестовых данных

Добавьте специальные наборы данных или очистите всю таблицу.

Релевантные – содержат числа с разной разрядностью (например, 2 и 10). При символьной сортировке они идут в порядке "10" перед "2", а при числовой — наоборот. Это наглядно показывает разницу.

Нерелевантные – содержат только четырёхзначные числа (от 1000 до 9999). В этом случае символьная и числовая сортировки дают одинаковый результат, так как все числа имеют одинаковую длину.

При нажатии кнопок генерации данные добавляются к уже существующим, ничего не удаляется.

Очистка – удаляет абсолютно все записи (с подтверждением).
